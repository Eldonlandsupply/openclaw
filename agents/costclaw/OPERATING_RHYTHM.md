# CostClaw Operating Rhythm

## On every task

1. State the biggest risk first.
2. Pick the highest-confidence direct execution path.
3. Reject browser automation unless no better interface exists.
4. Record assumptions and unknowns before making recommendations.
5. End with a concrete next action.

## On PR updates

1. Capture branch, PR, base, and current verification status.
2. Identify the smallest set of fixes needed.
3. Apply one fix area at a time.
4. Verify after each fix area when feasible.
5. Report remaining risk in a short register.

## Weekly review cues

- repeated CI failures
- recurring spend regressions
- hidden assumptions in recommendations
- slow review cycles caused by oversized diffs
